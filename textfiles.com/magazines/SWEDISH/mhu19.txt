 ·1 eller flera tändare (endast gasbehållaren) ELLER en co2-patron.

 * 8 * Javakurs ********************************************| Ze7th |*
 *********************************************************************
 
  Java-språket  är  ett  rätt  så nytt programmeringsspråk. Det är ett
 objektorienterat språk,  och det innebär att Java arbetar med klasser
 och  instanser  av  dessa s k objekt. Fördelen med objektorienterande
 språk  är  att  programmen  blir  (oftast)  stabilare och enklare att
 underhålla.  Java är väldigt likt C/C++ rent syntaxmässigt, men många
 saker  har tagits bort från C/C++ för att göra Java till ett säkrare,
 enklare  och  mer  objektorienterat  än vad C/C++ är. Java-språket är
 gjort  så att det ska kunna köras från i princip vilken plattform som
 helst.  Det du skriver in måste förstås av datorn annars så är det ju
 ingen mening med att programmera. Källkoden kompileras till maskinkod
 och kompileringen gör så att du får en körbar fil. Detta gör även att
 du  bara  kan köra filen från den plattformen som du kompilerat från.
 Detta  gäller  för  språk som t ex C/C++ och Pascal. Istället för att
 kompileras  som C-språken så interpreteras Java, dvs att man låter en
 "tolk" översätta källkoden till maskinkod varje gång programmet körs.
 Det  mest kända interpreterande språket är Basic och dess uppföljare,
 Visual Basic.  Nackdelen  med  interpreterande språk är bl.a. att man
 måste  ha  tillgång till utvecklingsmiljön eller filen där interpret-
 atorn ingår varje gång man vill köra programmet + att interpreterande
 program  blir  långsammare  p.g.a att det som du skrivigt (källkoden)
 måste översättas (maskinkod) varje gång programmet körs. 
 
 Java  har på bara några år fått rätt så bra tryck i seglen och blivit
 ett  av  dom  fräckare  språken, ett s k "hippie-språk". Att Java har
 blivit så populärt beror nog främst på att det passar så jävla bra på
 nätet.
 
 För  att  du  ska  kunna  greja med Java så måste du ha (som sagt) en
 kompilator  som omvandlar det du skriver till något ännu mera kufiskt
 som  datorn  begriper. Jag rekommenderar att du laddar ner en kompil-
 ator  (JDK)  från  sun  (http://java.sun.com)  som  är gratis/fri att
 använda  osv. Hur du får JDK att fungera borde inte vara några större
 problem, men om det är något som inte fungerar så kan du alltid maila
 till  mig. Du behöver en txteditor också (får man inte glömma). Denna
 kurs  kommer  till  största  del att handla om s k applets, dvs Java-
 program som man bäddar in i sin html-kod för att få en snygg hemsida.
 
 Okej, har du fått någon ordning på sakerna, skit samma för nu kör vi.
 Traditionsenligt så börjar vi med det enklaste programmet.
 --------------
 
 class mölj {
 }
 
 -------------
 Döp  filen  till "ovn1.java" (utan " "). Gå till dospromten och skriv
 för att kompilera texten:
 javac ovn1.java 
 
 Nu  om  allting  har  gått  rätt till och har du har förstått hur JDK
 fungerar,  så  har du en fil som heter mölj.class (mölj.class för att
 vi döpte klassen till mölj i programmet).
 Om  man  vill  att  klassen ska var tillgänglig  för andra klasser så
 anger man detta med ordet public. Då ska det se ut så här:
 -------------------
 
 public class mölj {
 }
 
 -------------------  
 När du gör detta så se till att namnet på klassen är detsamma som det
 namn du spara filen under.
 Om  du sparar klassen som x.java måste också den klass du skapar heta
 "x" (utan "").
 -----------------
 
 public class x {
 }
 
 -----------------
 OBS!  Du  måste  alltså spara filen med exakt samma namn som klassen,
 dvs filnamnet måste vara x.java i exemplet för att du ska kunna komp-
 ilera koden. Java GÖR skillnad på gemener och VERSALER vilket inte är
 värt att glömma!
 
 I  Java  finns  det  2  olika typer av program, dels fristående Java-
 program,  dels  Java-program som man bäddar in i html-kod. Den senare
 varianten  kallas  för  applets  och är kanske det vanligaste man hör
 talas  om.  Det  är  sådana applets som jag kommer att ta upp mest om
 eftersom  man  kan  göra  så  mycket till sin hemsida, så att den bli
 skitsnygg :).  För  att  bädda  in  en  applet  i html används taggen
 
 
 Taggen  
 
 Spara  html-koden  med  namnet x.html i samma mapp där du sparade den
 övriga  koden.  För  att  kolla hur det blev så är det bara att öppna
 skiten  i  en  webbläsare,  eller så kan du använda programmet Applet
 Wiewer som följer med i JDK.
 
 Detta program som vi har skapat idag fungerar tydligen, trots att det
 inte gör speciellt mycket förutom att existera. Du har nu (om du fått
 det  att  fungera..)  skapat  ditt första (troligtvis) program, eller
 applet.  Kom  ihåg att inte krångla till det med stora (VERSALER) och
 små  (gemener)  bokstäver  i  onödan  eftersom Java gör åtskillnad på
 dessa  *som sagt*,  och  det  innebär att det går åt helvete ifall du
 blandar.
 
 Man  kan  programmera  Java  direkt i html-koden, t ex göra en klocka
 (varför inte...) så här:
 
 ---------------------------------------
 
 
 
 
 
 
 
 Time is:
 
 
 
 
 
 
 ---------------------------------------
 
 Med  detta "java-in-html"  exempel vill jag bara visa att Java är ett
 flexibelt  språk,  dvs  att man inte är speciellt begränsad i VAD man
 gör och HUR man gör det.

 *********************************************************************
 
  Angående artikeln  om att hexeditera mirc, värt att nämnas är att om
 vill få en mer personlig look på i clienten mirc så rekommenderar jag
 för  den  som  inte orkar att sitta och ändra binär-data för hand att
 använda  en  editor  som  klarar  av att editera binary resources som
 bland annat Borland C++ (nyare än 4-nåt) gör. På så vis kan man ändra
 vad  i helvete man vill utan att ha själva källkoden till programmet.
 Flytta  runt  på knappar byta namn på knappar byta bilder/menyer osv.
 Hur man gör man då? Jo i Borland C++ så öppnar du filen genom att som
 vanligt  ta  File Open.. bläddra fram till önskad fil och markera din
 exe  som  ska  editeras.  Längst  ner ser du sedan att ordet "Viewer"
 syns,  välj  där  "Edit Binary Resources".  Prova  dig sedan fram att
 editera. Man kommer snabbt på vad som ändras. När du sedan är klar är
 det bara att spara filen, men du bör innan ha tagit en säkerhetskopia
 om  något  skulle gått fel. Detta funkar troligtvis också i MS visual
 C++.
