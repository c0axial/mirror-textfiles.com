     Fick SEH3 i min brevlåda, och jag blev mycket glad. Läste bland annat
     att Jan på Flashback hade länkat Zineworld. Mycket positivt och så
     men våra tidningar om vi får kalla dem det kommer nu säkerligen att
     belysas. Jag undrar varför. Ska man inte få skriva nåt utan att en
     myndighet måste läsa det? Vad ska de dra för slutsatser av vårt
     arbete? Ok. Det vi skriver om och lär ut är brottsligt. Det borde
     vem som helst fatta, men det finns en viktig skillnad här. Det enda
     vi gör, jag gör i alla fall är att vi bara skriver om det.
     Vi använder våra tangentbord för att bilda ord och meningar som
     beskriver dessa brott.
     Men hur skall man tolka det? Hjälper vi brottslingar att lära
     sig att begå nya brott eller är det så att vi kanske informerar
     allmänheten vad andra människor faktiskt kan ta sig till för att
     läsa/skada din Information? Tänkte väl det. Vi gör kanske samhället en
     tjänst. Vi avslöjar brottslingar, vi avslöjar brottslingars tillväga-
     gångsätt vilket hjälper dig att förebygga och skydda din Information.

     För i en värld av Information så är Informationen det enda som räknas...

     D I S C L A I M E R  [Ni vet nog vad vi menar...]


     Detta e-zine innehåller hacking och phreak relaterad information.
     Om du tycker att ovanstående ämnen verkar stötande skall du inte
     läsa vidare. Det är viktigt att förstå att denna tidning inte
     uppmanar dig till att begå brott. Du kan råka väldigt illa ut
     om du gör det som vi beskriver i våra artiklar, vi ber dig därför
     att inte göra det som vi beskriver. Om Du nu gör detta ändå kan
     vi inte stå till svars för dina handlingar. All text är till för
     utbildande och förebyggande syften. Det är väldigt bra att veta
     vad människor faktiskt kan göra för att skada dig och dina data.
     Vi bör också tillägga att vi själva inte använt oss av vad vi har
     skrivit mer än på våra egna system.

     Andra saker som vi som skriver denna tidning vill poängtera är
     att vi kommer att vara så neutrala som vi bara kan när vi
     skriver om andra grupper. Till skillnad från vissa andra elektroniska
     magazin kommer vi inte öppet kritisera någon annan grupp eller
     något annat zine. Vad vi däremot gärna gör är att gratulera och
     uppmärksamma zine som gör bra ifrån sig!

     Billigare mobilsamtal via Flashback (990828)-------------------- - -

     Flashback erbjuder nu en tjänst som innebär att man kan ringa billigare
     på sin mobiltelefon utan att man behöver byta abonnemang. Man behöver
     alltså inte byta telefonnummer. Tjänsten är ett så kallat mobilt
     accessabonnemang. Följande information är hämtad direkt från Flashbacks
     hemsida:


       * Upp till 40% billigare till fasta och mobila näten

       * Upp till 60% billigare till utlandet

       * Inga fasta avgifter (ingen månadsavgift)

       * Ingen uppkopplingsavgift (Telia tar 32 öre exklusive moms)

       * Du binder inte upp dig vid något långvarigt avtal

       * Specificerad faktura kostnadsfritt


     Genom att kombinera detta med ett kontantkort får man ett mycket
     billigt alternativ. Mer information och ansökningsformulär finns
     på adressen: http://www.flashback.net/gsm/

     [Avalon] (källa www.flashback.se)

     Bakdörrar i Windows? (990905)---------------------------------- - -

     Andrew Fernandes, chefstekniker på det amerikanska företaget Cryptonym
     hävdar att han har hittat två stycken nycklar som ger den Amerikanska
     underrättelsetjänsten NSA (National Security Agency) tillträde till
     alla nya versioner av Windows (Windows 95/98 och Windows NT/2000).

     En av de två nycklarna kan Microsoft använda för att ge obehöriga fulla
     rättigheter till Windows. Den andra nyckel är av okänd betydelse.
     Andrew Fernandes säger dock att namnet på den andra nyckeln är "_NSAKEY".

     Microsofts Säkerhetsansvarig, Scott Culp menar att nyckeln endast visar
     att operativsystemet är gjort efter NSA's säkerhetsnormer och
     tillbakavisar att Microsoft skulle lämna några bakdörrar i sina
     operativsystem.
     Datorklubben Kaos (Chaos Computer Club) hävdar att de också känt till
     nyckeln sedan en månad tillbaka.

     Visar det sig att dessa uppgifter är riktiga kommer troligtvis
     försäljningen av Windows 2000 få ett allvarligt bakslag. Enligt
     Amerikansk lag får inte NSA spionera på Amerikanska invånare men
     spionage på utländska människor finns det ingen lag på...

     Ännu en anledning till att krossa rutan och istället använda ett annat
     operativsystem. Valet är ditt.

     Läs mer: http://www.wired.com/news/news/technology/story/21577.html

     Publika och Privata nycklar

     Anta att Cyberdude vill skicka ett hemligt mail till HaxxorKid. Cyberdude
     letar då upp HaxxorKids publika nyckel och krypterar 'mailet' med hjälp
     av denna nyckel. Eftersom Haxxorkids publika nyckel inte kan dekryptera

     nyckel. Den enda som kan tyda mailet är HaxxorKid, som har den _Privata_

     och HaxxorKid inte behöver byta några nycklar med varandra, vilket skulle
     inneburit en risk om någon då skulle snappat upp nycklarna vid ett
     eventuellt utbyte via Internet.

     Det fiffiga systemet med publika nycklar kan också användas i andra syften,

     1.0 Historia i korthet

     ASP utvecklades för att det skulle vara mycket enklare att göra
     kraftfulla sidor med API (Application Programming Interface) eller
     Microsoft's  version ISAPI (Internet Server Application Programming
     Interface) Men der var inget bra med ISAPI, du måste vara bra på att
     programmera i C lr Perl och man behövde kompilera källkoden. Därför
     utvecklades ASP man lägger enkelt till ASP koden direkt i html koden.


     2.0 Servrar och sånt

     Det finns ett litet problem med det här med ASP du måste ha en server
     som stödjer ASP de som nu gör det är Microsoft's Personal Web Server
     (PWS) och Internet Information Server (IIS). PWS kan du köra på din Win
     95/98 burk, PWS 4.0 finns med på WIN98 cd'n och IIS finns till WIN NT
     Server lr Workstation 4.0 med Option Pack 4.0 installerat. Med du kan
     kan även köra ASP på andra Windows servrar med då behöver du installera
     ett plugin på servern det bästa som finns ute nu är ChiliASP som fungerar
     med de flesta popukära win servrar. Med det finns även
     plugin för vissa Unix servrar bla Apache mfl. En fördel med ASP är att
     det är mycket svårare att sno din källkod till programmen, nu vet jag
     att det fanns vissa buggar i vissa Servrar så att du kunde lägga till
     tex www.microsoft.com/default.asp;l tror jag att det var så fick du upp
     källkoden till asp sidan vilket inte var så bra..men nu finns det nog
     inga servrar som kör asp och har kvar den buggen..men testa ni.Men åter
     till ämnet nu..När din webläsare frågar webservern efter en sida så tar
     han och läser igenom dokumentet efter nån asp taggar som han kan köra
     hittar han några så kör han dem och resultatet skickar han till browsern
     som bara ser resultatet av körningen av koden tex..

     <html>
     <body>

     [ NT säkerhet ] av rentor ______________________________________ _ _


     FILSYSTEM

     Microsoft Windows Nt server har stöd för olika filsystem:
     FAT, NTFS och CDFS.

     FAT - är av samma konstruktion som operativsystemet DOS och medger
     att du kan ha NT och DOS på samma partition.
     du kan inte använda NT's avancerade säkerhetsfunktioner med FAT
     (avancerade my ass).

     NTFS - (NT File System) är konstruerat speciellt för NT. All
     läsning och skrivning övervakas ständigt av systemet. NTFS är
     mycket säkrare både mot intrång och haveri än FAT.

     CDFS - är ett filsystem enbart anpassat för läsning av cdrom.
     innehåller också ett cacheminne speciellt anpassat för cdrom.

     --------------------------------------------------------------------
     --------------------------------------------------------------------

     DOMÄNER

     när du installerar NT så har du tre alternativ att välja för
     serverns roll i domänen.

     PDC, BDC, SERVER.

     PDC - (Primary Domain Controller) är den server som styr behörighet och
     användare i domänen. det kan bara finnas en PDC i domänen.

     BDC - (Backup Domain Controller) är en server som avlastar PDC genom
     att svara på inloggningar från klienter.

     SERVER - den står utanför domänhanteringen. Serven kan användas till resurser
     som alla måste nå, tex email osv.

     --------------------------------------------------------------------
     --------------------------------------------------------------------

     PASSWORDFILEN

     NT's passwordfil kan gemföras unix etc/passwd , fast här heter den
     /winnt/system32/sam. den är inte speciellt svårcrackad. det krävs två
     program: samdump och en password cracker, och helst en fet ordfil.

     samdump - gör om filen till en textfil, sen är det bara till att cracka.

     Samdump och lämpligt bruteforce program hittar du hos L0pht.
