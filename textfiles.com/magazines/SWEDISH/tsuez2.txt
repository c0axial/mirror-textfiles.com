07. Cracka Windows Password List (PWL)

Windows har ingen bra säkerhets nivå enligt diverse artiklar som hittas på
Internet. Dettas stämmer totalt, enligt min åsikt.

Hur får man tag i en .pwl fil ?
Det enklaste är Social Enigeering på IRC. Annars kan du andvända en Trojan
så du får access till offret som andvänds för att få tag i .pwl filen.
Men detta anses som lamt. Så det är ingen att rekommendera.

Hur crackar man en .pwl ?
Antag att du andvänt Social Enigeering för att få tag i en och du har fått
tag i en som heter lemer.pwl.
Flytta lemer.pwl till C:\WINDOWS
Logga sedan in som en annan andvändare och skriv in "lemer" som login och
valfritt som password. Nu kommer en fil i C:\WINDOWS skapas som heter
i stil med lemer.000. Låt den vara så länge. Ta nu bort lemer.pwl och sen
så döper du om lemer.000 till lemer.pwl. Kör sedan en .pwl cracker på
.pwl filen. En sådan cracker finns överallt på Internet. Dom 2 bästa skulle
kunna vara Win95 PWL viewer och glide.exe.
Om du väljer glide.pwl skriv <glide.exe lemer.pwl lemer.txt> vid DOS prompten.

08. Simpelt lösenords skydd

System krav: Windows 95, DOS

Jag gjorde ett simpelt litet lösenord skydd för dos. Inte alls svårt att knäcka,
men passar bra som snabbkommando i alla fall.
Antag att du har en mapp på plats: c:/psycho/misc/html/pics som innehåller bilder
på nakna flickor. Låt oss säga att din farsa/morsa inte skulle gilla att du har
pornografiskt material. Gå ut i DOS och starta edit och skriv sedan.

@echo off
explorer c:/psycho/misc/html/pics

Spara filen som valfritt.bat i en katalog som dina föräldrar inte brukar pilla
i, jag skulle ha sparat den i c:/Windows/System
Eftersom /system är en patchad mapp så kan den nås från var som helst på
Start/Kör och i dos skalet. Sen döljer du mappen /pics och dölj även
valfritt.bat också. Sen är det bara att skriva valfritt på kör på
Startmenyn så startas din lilla bat fil och därmed öppnas /pics mappen.
Detta är inte alls säkert, men du kan även andvända det som ett snabb
kommando system för att gå till mappar och starta program m.m.

-==- FÖRORD -==-
Java är ett nytt objektorienterat språk som många hävdar ska
bli framtidens programmeringsspråk. Och visst har det stora
fördelar jämfört med äldre språk. Men också en del nackdelar.
Java är en direkt efterföljare till C++ som i sin tur bygger
på C. Vilket märks. Mycket av det som finns i Java är hämtat
från dessa två språk och en skicklig C++ programmerare kommer
att ha lite (om några) problem med att lära sig Java. Men
med Java ville man gå lite längre. När den globala
kommunikationen inte längre var en framtidsvision ville man
ha ett språk som var platformsoberoende och som skulle fungera
lika bra på vilket operativsystem du än körde det på. Man
ville också att det skulle vara enkelt uppbyggt så att
programmerare som tidigare arbetat med andra språk, snabbt
skulle lära sig och flytta över sin kunskap och erfarenhet
till Java. Till stor del har man lyckats. Men det har också
bidragit med en del nya nackdelar. Under utvecklandet av
Java dök det upp en ny faktor som faktiskt skulle påverka
utvecklandet och ge en Java en stabil bas att stå på. Det
var World Wide Web som under samma tid hade en explosionsartad
tillväxt. Under vilket annat system skulle passa mer för Java?
Internet är ju ett enormt nätverk av olika CPU, OS odyl. 
Vilket gjorde att implementation för just World Wide Web vart
en viktig faktor för Java. Detta ledde bland annat fram till
skapandet av så kallade applets. Dessa skulle kringå de
tidigare problem man haft med kommunikation och säkerhet som
andra språk haft. Tidigare var man tveksam till att ladda hem
program med tanke på virus, trojaner och informationssamlare,
men Java har lyckats bygga upp en effektiv "firewall" mellan
programmet och klientdatorn. Javas portabilitet kommer från
det som kallas bytecode. I traditionella språk kompileras 
programmet till maskinkod som sedan exekveras. Java däremot
har en interpeter istället för en kompilator vilket gör att
koden aldrig blir låst till en platform. Interpetern ser till
att tolka bytecoden till just det system som är aktuellt. 
Många ser Java som utökad HTML, men så är absolut inte fallet.
Det är egentligen bara den lilla sektionen applets som direkt
har med det att göra, men det finns mycket mer i språket Java.

  --==-- Skillnader mellan Java & C++ --==--
Främsta skillnaden är som tidigare nämnts att C++ program
kompileras medan Java program "tolkas". Förutom det finns
det också skillnader inom språkenas arkitektur. I Java
är allting (förutom variabler) uppbyggt kring klasser och
metoder (Javas namn på funktioner). Detta för för att stödja
de gamla metoderna Polymorphism, Encapsulation och Inheritance
som började användas i det objektorienterade C++. Men till
skillnad från C++ är Java striktare inom säkerheten. Klass
medlemmar skyddas, ingen minneshantering tillåts, lokal påverkan
på filer och annan data är förbjuden och så vidare. Vilket
har gjort Java till ett mycket säkert men kanske lite 
tillbakhållet språk. Låt mig ge de största skillnaderna:

   * Pekare är ärftligt osäkra. Tillåts ej.

   * Java har inte stöd för strukturer eller unions.

   * Java tillåter inte operator overloading.

   * Java stödjer inte preprocessor direktiv.

   * Java utför inga automatiska typkonverteringar.

   * All kod i Java är inkapslad i en eller fler klasser
     vilket gör att det inte finns globala variabler eller 
     globala funktioner i det hänseendet. 

   * Java tillåter inte default argument.

   * Java tillåter inte inheritance av flera super klasser
     till en subklass.

   * Även om Java har konstrukterare har den ingen destruktor,
     utan har istället lagt till funktionen finalize().

   * Java stödjer inte typedef.

   * Java stödjer inte unsigned integers.

   * Java tillåter inte goto.

   * Java har ingen delete operator.

   * << och >> är i Java inte overloaded för IO-funktioner.

   * I Java är objekt skickade per referens endast. I C++ kan dom
     skickas via värde eller referens.

Java har dessutom lagt till lite nya funktioner. 

   * Javas multithread är enklare och effektivare.

   * C++ har ingenting som kan jämföras med Javas package.

   * interface är en renare represantation av C++ abstrakta klasser.

   * Java har en mer riktad användning av minne. Liksom C++ har den
     funktionen new, Java har däremot ingen delete.

   * Java ersätter C++ standardbibliotek och erbjuder en helt ny
     uppsättning API klasser. Där C++ standardbibliotek bara var
     en liten del objekt-orienterad är Javas det helt och hållet.

   * break och continue har förbättrats.

   * char typen i Java är automatisk en 16-bit-wide-Unicode, liknande
     wchar_t i C++.

   * Java har lagt till >>>, vilken gör en unsigned right shift.

   * Förutom enrads-kommentarer och flerrads-kommentarer har Java också
     dokument-kommentarer. 

   * I Java är String inte en typdeklaration utan en klass. Ungefär som
     string klassen i C++.

Dessutom finns det tre områden som båda understödjer men som skiljer sig något.

   * Både C++ och Java stödjer Boolean typer, men Java har ingen numerisk
     motsvarighet till true och false (i C++ är true icke-noll, false noll).

   * När du skapar en klass i C++ så gäller access specificierarna en grupp
     statements. I Java gäller de bara för deklarationerna de omedelbart
     företräder.

   * C++ stödjer också exception handling, men i C++ är det inget krav att
     en exception blir fångad.

För att ha någon glädje över den här lilla exempelkoden behöver du någon av

Till skillnad från andra programspråk är det i Java viktigt vad du döper
filen med koden till. I det här exemplet döp den till HelloWorld.java.

/*

*/

   }

Med JDK skriver du nu 'javac HelloWorld.java' för att skapa .class filen.
Sedan skriver du 'java HelloWorld'. Coolt eller hur?
